# Contributing to LANCache Manager

Thanks for your interest in contributing. Bug reports, fixes, translations, and
features are all welcome.

## How to contribute

1. Open an issue first for anything non-trivial so we can agree on the approach.
2. Fork the repo and create a branch for your change.
3. Keep changes focused and include a clear description of what and why.
4. Make sure the project builds and lints before opening a pull request.

## Developer Certificate of Origin (DCO)

We use the [Developer Certificate of Origin](https://developercertificate.org/).
It is a simple statement that you wrote the contribution or otherwise have the
right to submit it under the project's license. To sign off, add a line to each
commit message:

```
Signed-off-by: Your Name <your.email@example.com>
```

You can do this automatically with `git commit -s`. The sign-off certifies that
you agree to the DCO for that contribution.

This keeps the project's licensing clear and lets the project adopt future
license versions without having to track down individual contributors.
